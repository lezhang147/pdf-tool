> 版本：v1.2 | 日期：2026-07-08 | 作者：数妙妙  
> 产品定位：闲鱼渠道分销的轻量级 PDF 在线编辑工具，通过工具化替代人工服务实现高利润变现

## 1. 产品概述

### 1.1 产品一句话描述

一个浏览器内运行的 PDF 在线编辑工具，用户上传 → 在线改字/去水印/合并拆分 → 下载，全部在本地浏览器完成，无需上传服务器，发个链接就能用。

### 1.2 核心价值主张

| 对比维度 | 闲鱼手工服务（现状） | PDF在线工坊（我们） |
|---------|-------------------|------------------|
| 客单价 | 1.8元/次 | 1.8-29元 |
| 交付方式 | 来回沟通+传文件+等待 | 上传→操作→下载，秒级出结果 |
| 隐私安全 | 文件发给陌生人，有泄露风险 | 浏览器本地处理，文件不出电脑 |
| 边际成本 | 每单都需人工操作 | 零，卖1万份跟卖1份成本一样 |
| 卖家收益 | 接50单=90元 | 卖10单永久卡=290元 |

### 1.3 目标用户

- **主要用户**：在闲鱼搜索"PDF改字""PDF去水印""PDF合并"的普通用户
  - 大学生（改论文格式、处理扫描件）
  - 职场人（修改合同、处理商务文件）
  - 小微企业主（处理报价单、证明材料）
- **使用场景**：手机或电脑上临时需要处理 PDF，不想装大软件（如 Adobe Acrobat），也不想花钱找人工

### 1.4 竞品分析

| 竞品 | 模式 | 价格 | 弱点 |
|------|------|------|------|
| Smallpdf / iLovePDF | 海外在线工具 | 免费有限制/付费订阅 | 需要网络上传文件到服务器，国内访问慢，隐私顾虑 |
| Adobe Acrobat | 桌面软件 | ¥150+/月 | 太重，手机用不了，贵 |
| WPS PDF | 国产桌面软件 | 会员 ¥89/年 | 需安装，功能绑定WPS全家桶 |
| 闲鱼人工改PDF | 手工服务 | 1.8-50元/次 | 慢、隐私风险、沟通成本高 |

## 2. 功能需求

### 2.1 V1.0 MVP 功能（第1-2周上线）

#### F1 — PDF 上传
- **优先级**：P0（必须有）
- **描述**：用户选择本地 PDF 文件或拖拽上传
- **细节**：支持点击按钮或拖拽，大小限制 50MB，仅支持 .pdf 格式。

#### F2 — PDF 页面预览
- **优先级**：P0
- **描述**：在页面中渲染 PDF 内容，支持翻页
- **细节**：基于 PDF.js 在 Canvas 渲染，支持移动端双指缩放和桌面端滚轮缩放。

#### F3 — 在线编辑文字（覆盖模式）
- **优先级**：P0
- **描述**：在 PDF 页面上添加或修改文字内容
- **细节**：
  - **核心逻辑**：采用“白底矩形覆盖原字 + 添加新文本框”的涂改液逻辑，而非原生段落编辑（需在 UI 和闲鱼文案中明确预期，防止客诉）。
  - 支持拖拽移动文本框，修改字号、颜色及常用中文字体。

#### F4 — 额度核销与导出下载（核心交易节点）
- **优先级**：P0
- **描述**：用户点击下载时，根据额度状态决定导出的 PDF 质量。
- **细节**：
  - **无额度状态（未激活）**：允许下载，但全篇强制添加半透明的“未激活”网格水印，供用户确认修改效果。
  - **有额度状态**：
    1. 用户点击下载，前端向 Worker 发起核销请求（`/consume`）。
    2. Worker 验证卡密，扣除 1 点额度，**并精准记录消耗日志（时间戳、IP、动作）**。
    3. Worker 返回核销成功，前端生成无水印的清晰 PDF 并触发浏览器下载。

#### F5 — 积分/额度卡密系统
- **优先级**：P0（商业必要性）
- **描述**：采用「通用卡密（License Key）+ 额度系统」管理用户权限。
- **交付物**：一串激活码，例如 `PDF-8F2A-93BC`
- **激活流程**：
  1. 买家在闲鱼拍下，通过自动发卡网提取卡密。
  2. 买家在任意设备（电脑/手机）打开网页，输入卡密。
  3. 前端将卡密存入 `localStorage`，直接从远端读取该卡密的剩余额度。
  4. **跨端优势**：一个卡密支持手机和电脑同时使用，共享额度池，极大降低正常用户的摩擦。

## 4. 技术方案

### 4.1 技术选型
- **前端**：Next.js 16 + React 19 + Tailwind CSS 4
- **PDF引擎**：pdfjs-dist 4.0 (渲染) + pdf-lib (纯前端修改保存)
- **防篡改**：`javascript-obfuscator` (前端打包混淆，防止拦截核销请求)
- **后端**：Cloudflare Workers + KV (免费额度够用，轻量级卡密与日志管理)

### 4.2 额度核销与日志架构

```text
┌─────────────────────────┐
│        Browser          │
│                         │
│ [点击 无水印下载]       │
│         │               │
│         ▼               │
│ 向后端发起核销请求      ├───( HTTPS )───▶ ┌──────────────────────┐
│ { key: "PDF-...",       │                 │ Cloudflare Worker    │
│   action: "download" }  │                 │                      │
│         │               │                 │ 1. 查 KV 余额         │
│         ▼               │                 │ 2. 扣减 1 次额度      │
│ 收到成功响应，开始本地  ◀───(返回成功)────┤ 3. 记录消耗日志       │
│ 使用 pdf-lib 组装文件   │                 │   (精确时间, IP, 动作)│
│         │               │                 └──────────────────────┘
│         ▼               │
│ 触发 Blob 下载保存      │
└─────────────────────────┘
```
*(注：PDF文件绝不上传，保护隐私。网络请求只传递几十字节的核销指令与卡密)*

### 4.3 Cloudflare Worker 关键代码示例

```javascript
// worker/index.js (核心核销逻辑)
export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    
    // 核销额度接口
    if (url.pathname === '/consume' && request.method === 'POST') {
      const { licenseKey, action } = await request.json();
      
      const keyData = await env.KV.get(`key:${licenseKey}`, { type: "json" });
      if (!keyData) return Response.json({ error: '无效卡密' }, { status: 403 });
      if (keyData.points <= 0) return Response.json({ error: '额度已用完' }, { status: 403 });
      
      // 扣减额度
      keyData.points -= 1;
      await env.KV.put(`key:${licenseKey}`, JSON.stringify(keyData));
      
      // 重点：记录消耗日志（作为闲鱼小法庭铁证）
      const clientIP = request.headers.get('CF-Connecting-IP');
      const timestamp = new Date().toISOString();
      const logEntry = `${timestamp} | IP:${clientIP} | Action:${action} | 剩余额度:${keyData.points}`;
      
      // 将日志追加到该卡密的记录中
      let logs = await env.KV.get(`logs:${licenseKey}`) || "";
      logs += logEntry + "\n";
      await env.KV.put(`logs:${licenseKey}`, logs);
      
      return Response.json({ success: true, remaining: keyData.points });
    }
    
    return Response.json({ error: 'Not found' }, { status: 404 });
  }
};
```

## 6. 闲鱼防白嫖退款体系（重构版）

> **真实痛点**：闲鱼虚拟商品交易中，最大的风险是买家完成了工作需求，随即以“不好用”等理由申请退款（用完即退）。
> **防御核心**：不再折腾正常用户的设备绑定，转而**在后端构建不可抵赖的“额度消耗凭证”**，专攻闲鱼小法庭举证环节。

### 6.1 第一层：话术与流程预期管理

在自动发卡或聊天发货时，必须带上明确的声明：
> “亲，您的卡密已发送。本工具为在线服务，体验版可免费确认效果。**凭卡密点击下载成品会扣除相应额度，额度一经消耗即代表服务已提供，不支持无理由退款哦**。遇到操作问题可随时咨询。”

### 6.2 第二层：消耗日志铁证（专治“用完即退”）

当遇到买家“用完即退”且胡搅蛮缠时：
1. **调取后台日志**：卖家在 Worker 管理后台（或自制面板）提取该买家卡密的 `logs` 记录。
2. **小法庭强力举证**：将日志截图（包含精准的时间戳）提交给闲鱼客服/小法庭。
3. **高胜率举证话术**：
   > “买家属于恶意白嫖退款。体验版完全免费且能预览所有效果，买家确认没问题后才输入的卡密。
   > 根据系统后端额度核销日志（见图），买家于 `2026-xx-xx 15:30:12` 成功核销了额度并下载了成品 PDF。
   > **我们的虚拟服务已经履行完毕，买家已经完成了他的工作需求，现在是用完即退。** 请小法庭明鉴并驳回退款。”

**实战效果**：闲鱼小法庭评审员面对这种包含准确时间戳的“后台日志硬证据”，通常会认定买家已享受了服务，从而极大提升卖家的胜诉率。

### 6.3 第三层：远程废除止损

如果遇到了极个别的极端判罚导致退款成功，卖家可通过后台将该卡密的额度直接**清零**。
买家无论在手机还是电脑上，再次点击下载时，Worker 将拒绝核销并报错，及时切断损失。
